# sool 서비스 분석 보고서

본 문서는 전통주 데이터 플랫폼 **sool** 프로젝트의 구조와 코드를 분석하여 서비스의 목표와 주요 타겟을 정리한 보고서입니다.

## 1. 프로젝트 개요
**sool**은 한국 전통주의 데이터를 수집, 분석하고 시각화하여 제공하는 플랫폼입니다. 단순히 정보를 나열하는 것을 넘어, 제품의 풍미 지표를 데이터화하여 사용자별 맞춤형 경험과 비즈니스 인사이트를 제공합니다.

- **기술 스택**: FastAPI (Backend), Next.js (Frontend), MariaDB (Database)
- **주요 데이터**: 제품 기본 정보, 감각 노트(Sense), 테이스팅 노트, 리뷰 등

## 2. 주요 타겟 및 핵심 가치

### 2.1 양조장 (Breweries)
양조장은 자사 제품의 시장 내 위치를 파악하고 품질 개선을 위한 데이터를 얻을 수 있습니다.
- **핵심 가치**: 시장 포지셔닝 분석, 제품 데이터 수집 및 피드백 확인.
- **관련 기능**:
    - **제품 분석 리포트 (`report_generator.py`)**: 전체 DB 대비 해당 제품의 맛 지표(단맛, 산미 등)를 백분위와 레이더 차트로 시각화.
    - **시장 데이터 대시보드**: 카테고리별/지역별 분포 데이터를 통한 트렌드 파악.

### 2.2 전통주 취급점 (보틀샵, 주점)
전통주를 판매하는 비즈니스 파트너는 고객 상담과 큐레이션의 근거 자료로 플랫폼을 활용합니다.
- **핵심 가치**: 전문적인 큐레이션 콘텐츠 확보, 재고 및 제품 정보 관리.
- **관련 기능**:
    - **탐색(Explore) 및 필터링**: 지역, 카테고리, 도수별 상세 검색 기능.
    - **데이터 시각화**: 고객에게 제품의 풍미 특징을 객관적인 차트로 제시 가능.

### 2.3 전통주 소비자 (애호가)
일반 소비자는 자신의 취향을 발견하고 기록하는 개인화된 경험을 얻습니다.
- **핵심 가치**: 개인 시음 기록 관리, 취향 기반 제품 추천.
- **관련 기능**:
    - **테이스팅 노트 (`TastingNote`)**: 자신의 시음 느낌을 향, 단맛, 산미, 바디감 등으로 정밀하게 기록.
    - **추천 시스템 (`recommender.py`)**: 풍미 벡터(Flavor Vector) 분석을 통한 유사 제품 추천.

## 3. 시스템 구성 요소 분석

### 3.1 백엔드 (FastAPI)
- **API V2 설계**: 구버전과의 호환성을 고려하면서도 확장성 있는 REST API 제공.
- **데이터 분석 서비스**: Pandas, Matplotlib, Seaborn을 활용하여 서버 사이드에서 직접 분석 리포트(이미지/Base64) 생성.
- **보안 및 인증**: JWT 기반 인증 시스템으로 사용자별 테이스팅 노트 및 관리 기능 보호.

### 3.2 프론트엔드 (Next.js)
- **반응형 대시보드**: Ant Design과 Recharts를 활용하여 관리자와 일반 사용자에게 최적화된 시각적 UI 제공.
- **이미지 최적화**: 로컬 및 원격 이미지를 유연하게 처리하는 이미지 리졸버 구현.

## 4. 결론 및 제언
**sool** 프로젝트는 전통주 산업의 디지털 전환(DX)을 지원할 수 있는 강력한 데이터 기반을 갖추고 있습니다. 특히 양조장과 판매처를 잇는 B2B 성격과 소비자의 기록을 담는 B2C 성격이 공존하여, 향후 전통주 생태계의 허브 역할을 할 잠재력이 큽니다.
